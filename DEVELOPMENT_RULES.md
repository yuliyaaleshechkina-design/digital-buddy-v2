# Правила разработки для Digital Buddy

## 🚨 Главные правила

### 1. НИКОГДА не удаляй существующую функциональность
- Если нужно изменить функцию - **модифицируй**, а не удаляй
- Перед удалением любого кода - **запусти тесты**
- Если удалил что-то по ошибке - **сразу откатывай** (`git restore` или проверь что было)

### 2. ВСЕГДА тестируй перед сдачей
```bash
# Запуск всех тестов
python test_full.py

# Или через pytest (если установлен)
pytest test_full.py -v
```

### 3. Перед изменением файла:
1. **Прочитай** текущий код
2. **Пойми** что он делает
3. **Проверь** где он используется
4. **Измени** минимально необходимое
5. **Протестируй** что ничего не сломал

### 4. При добавлении новой функции:
1. Добавь **тесты** для новой функции
2. Проверь что **старые тесты** проходят
3. Обновь **README** если нужно

---

## 📋 Чеклист перед коммитом/сдачей работы

- [ ] Запустил `python test_full.py` - все тесты проходят
- [ ] Проверил что **старые функции** работают (не сломал при изменениях)
- [ ] Новая функция имеет **тесты**
- [ ] Обновил **документацию** если изменился API
- [ ] Проверил в браузере что UI работает

## 🔧 Частые проблемы и решения

### Проблема: "Удалил функцию по ошибке"
**Решение:**
```bash
# Посмотри что было в файле
git diff HEAD~1 app.py

# Восстанови файл
git restore app.py
```

### Проблема: "Тесты не проходят после изменений"
**Решение:**
```bash
# Запусти тесты с подробным выводом
python test_full.py -v

# Найди какой тест упал и исправь
```

### Проблема: "Добавил функцию но сломал другую"
**Решение:**
1. Посмотри **стек трейс** ошибки
2. Найди **какой файл** изменил
3. **Отмени изменения** в этом файле
4. Сделай изменения **осторожнее**

## 🧪 Тесты

### Что тестируется:
1. **AI функции** - анализ сентимента, генерация ответов, алерты
2. **БД функции** - добавление/получение новичков, сообщений, настроений
3. **Интеграция** - полный поток работы новичка

### Как добавить новый тест:
```python
def test_my_new_function(self):
    """Краткое описание"""
    result = my_new_function(input)
    self.assertEqual(result, expected)
```

### Запуск тестов:
```bash
# Все тесты
python test_full.py

# Один конкретный тест
python -m unittest test_full.TestAISentiment.test_positive_sentiment

# С подробным выводом
python test_full.py -v
```

## 🐳 Docker

### Запуск:
```bash
# Сборка и запуск
docker-compose up -d --build

# Только запуск (если уже собрано)
docker-compose up -d

# Остановка
docker-compose down

# Логи
docker-compose logs -f
```

### Обновление кода в контейнере:
```bash
# Копируем файлы в контейнер
docker cp app.py digital-buddy-app-1:/app/app.py
docker cp utils/database.py digital-buddy-app-1:/app/utils/database.py

# Перезапускаем
docker restart digital-buddy-app-1
```

## ✅ Правила для AI (меня)

### ВСЕГДА:
1. **Читать** файл перед изменением
2. **Понимать** что делает код
3. **Сохранять** существующую функциональность
4. **Добавлять** тесты для новых функций
5. **Проверять** что старые тесты проходят
6. **Сообщать** если что-то потенциально ломается

### НИКОГДА:
1. Не удалять функции без явного запроса
2. Не менять API без уведомления
3. Не удалять тесты
4. Не игнорировать ошибки тестов

### Если нужно изменить большую функцию:
1. **Предупредить** пользователя
2. **Показать** что будет изменено
3. **Получить согласие**
4. **Добавить тесты** до изменений
5. **Проверить** что всё работает после

## 📞 К кому обращаться если что-то сломалось

1. **Тесты не проходят** - смотри `test_full.py`, ищи падающий тест
2. **UI не работает** - смотри `app.py`, проверяй консоль браузера (F12)
3. **БД не работает** - смотри `utils/database.py`, проверяй `data/digital_buddy.db`
4. **Docker не работает** - `docker-compose logs`, проверяй `.env`
